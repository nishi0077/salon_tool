## 2. 課題の共感

### 見出し

- 施術中に電話やLINEに出られず、予約の取りこぼしが起きる

- 予約の変更・キャンセル対応に、毎回手がかかる

- スタッフごとに案内の質がばらつく

### Q1. 今使っている予約システムと併用できますか？

**A.** はい。FUNECTは「受付業務の自動化・補助」を担うレイヤーとして設計しています。既存の予約台帳や導線と役割を分けて導入いただく想定です。（連携の具体仕様はお問い合わせ時にご案内します。）

**A.** 施術に関するクレームや、想定外の問い合わせなど、リスクが高い内容は**AIが勝手に断定せず、スタッフへ引き継ぎ**ます。現場の安心感を優先した設計です。

**A.** チャットを中心とした受付に対応します。詳細な対応チャネルは資料またはお問い合わせでご確認ください。

# ワイヤーフレーム指示

## 全体

- **レイアウト**: 1カラム縦長。最大幅 **1140〜1200px** 中央寄せ。セクション間は **80〜120px**（モバイルは **56〜80px**）の余白。  
- **ブレークポイント**: デスクトップ基準 **1200px**、モバイル **375px** でワイヤー確認。  
- **タイポ階層**: H1（ヒーロー1つのみ）→ H2（セクション）→ H3（小見出し・表の列など）→ 本文 / 箇条書き。  

## セクション別：要素階層と配置

### 1. ヒーロー

| 要素 | 指示 |
|------|------|
| H1 | 案A or B のいずれか1つ（A/Bテスト時は別URLまたはツールで） |
| サブ | H1直下、最大2〜3行 |
| 補足 | サブ下、文字サイズ1段階小さく、行間広め |
| バッジ／帯 | H1上またはヒーロー最下部のフルワイド帯（「2週間トライアル」「月額9,800円（税抜）」） |
| CTA | Primary（塗り）＋ Secondary（枠線）を横並び → **375pxでは縦積み、Primaryを上** |
| ビジュアル | 右カラム or ヒーロー下：チャットUIモック1枚 **または** アイコン3点（24h / LINE / 人へ引き継ぎ） |

### 2. 課題の共感

| 要素 | 指示 |
|------|------|
| H2 | セクション見出し |
| リード | 2〜3行 |
| リスト | 7項目。アイコンは統一（×や!など）。**モバイルは左揃え、行間ゆったり** |
| 締め文 | リスト直下、太字1フレーズ |

### 3. 解決（FUNECT）

| 要素 | 指示 |
|------|------|

### 4. 比較軸

| 要素 | 指示 |
|------|------|
| H2 | セクション見出し |
| リード | 2行 |
| 表 | **デスクトップ**: 1行目に比較軸6列＋先頭列に行ラベル。**モバイル**: (a) 横スクロール＋「横にスワイプ」ヒント、(b) または **アコーディオン**（比較軸ごとにFUNECTと「一般的な〜」の2値） |
| 注釈 | 表下、小さめ文字で免責・FAQ誘導 |

**表の列順（固定）**  
1. 比較軸名（行見出し列）  
2. 一般的な予約システム列  
3. LINE＋簡易ボット列

※「◎○△」やチェックマークで視覚化。FUNECT列のみ背景を薄いブランド色で強調可（コントラスト注意）。
